## Overview

Status endpoints provide system information, maintenance operations, data export/import, and search functionality across families, samples, and functions.

---

## Get Welcome Message

<RequestExample>
```bash cURL
curl -X GET https://your-mcrit-instance/
```
</RequestExample>

<ResponseExample>
```json Response
{
  "status": "successful",
  "data": {
    "message": "Welcome to MCRIT"
  }
}
```
</ResponseExample>

---

## Get Status

<ParamField query="with_pichash" type="boolean" default="false">
  Include PicHash statistics in the status response
</ParamField>

<RequestExample>
```bash cURL
curl -X GET "https://your-mcrit-instance/status?with_pichash=true"
```
</RequestExample>

<ResponseExample>
```json Response
{
  "status": "successful",
  "data": {
    "samples": 1234,
    "functions": 56789,
    "families": 42,
    "pichash_index_size": 12345,
    "minhash_index_size": 67890
  }
}
```
</ResponseExample>

Returns current system statistics including number of samples, functions, families, and optionally PicHash index information.

## Export Data

<ParamField query="compress" type="boolean" default="false">
  Compress the exported data
</ParamField>

<RequestExample>
```bash cURL
curl -X GET "https://your-mcrit-instance/export?compress=true"
```
</RequestExample>

<ResponseExample>
```json Response
{
  "status": "successful",
  "data": {
    "samples": {...},
    "families": {...},
    "functions": {...}
  }
}
```
</ResponseExample>

Exports the entire MCRIT database. Use the `compress` parameter to reduce payload size.

<Note>Large databases may exceed memory limits. Consider using the selective export endpoint for partial exports.</Note>

---

## Export Selection

<ParamField path="comma_separated_sample_ids" type="string" required>
  Comma-separated list of sample IDs to export (e.g., "1,2,3,42")
</ParamField>

<ParamField query="compress" type="boolean" default="false">
  Compress the exported data
</ParamField>

<RequestExample>
```bash cURL
curl -X GET "https://your-mcrit-instance/export/1,2,3?compress=true"
```
</RequestExample>

<ResponseExample>
```json Response
{
  "status": "successful",
  "data": {
    "samples": {...},
    "families": {...},
    "functions": {...}
  }
}
```
</ResponseExample>

Exports only the specified samples and their associated data.

## Import Data

<RequestExample>
```bash cURL
curl -X POST https://your-mcrit-instance/import \
  -H "Content-Type: application/json" \
  -d @export_data.json
```
</RequestExample>

<ResponseExample>
```json Response
{
  "status": "successful",
  "data": {
    "samples_imported": 42,
    "functions_imported": 1234,
    "families_imported": 5
  }
}
```
</ResponseExample>

Imports previously exported MCRIT data. The request body should contain the JSON export data structure.

## Search Families

<ParamField query="query" type="string" required>
  Search term to match against family names
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor for fetching next page of results
</ParamField>

<ParamField query="limit" type="integer" default="100">
  Maximum number of results to return
</ParamField>

<ParamField query="sort_by" type="string">
  Field to sort results by
</ParamField>

<ParamField query="is_ascending" type="boolean" default="true">
  Sort order (true for ascending, false for descending)
</ParamField>

<RequestExample>
```bash cURL
curl -X GET "https://your-mcrit-instance/search/families?query=malware&limit=50&is_ascending=false"
```
</RequestExample>

<ResponseExample>
```json Response
{
  "status": "successful",
  "data": {
    "results": [
      {
        "family_id": 42,
        "family_name": "malware_family_1",
        "sample_count": 15
      }
    ],
    "cursor": "next_page_token",
    "total": 127
  }
}
```
</ResponseExample>

Search for families matching the query term.
