## Overview

MCRIT uses MinHash for efficient fuzzy function matching. The `MinHashConfig` class controls hash generation strategy, signature size, matching thresholds, and performance tuning.

## MinHash Strategy

<ParamField path="MINHASH_STRATEGY" type="MinHasher constant" default="MinHasher.MINHASH_STRATEGY_SEGMENTED">
  MinHash calculation strategy to use. This determines how function code is processed for hashing.
</ParamField>

<ParamField path="MINHASH_TRACK_SHINGLES" type="bool" default="False">
  Store the combination of shingles (unsorted) that were used to create each MinHash.
  
  <Note>
  Enabling this increases storage requirements but provides additional debugging information.
  </Note>
</ParamField>

## Function Filtering

<ParamField path="MINHASH_FN_MIN_INS" type="int" default="10">
  Minimum number of instructions a function must have to be considered for MinHashing.
  
  Functions below this threshold are too small to produce meaningful hashes.
</ParamField>

<ParamField path="MINHASH_FN_MIN_BLOCKS" type="int" default="0">
  Alternative minimum based on basic block count. A function must have at least this many basic blocks to be considered for MinHashing.
  
  Set to `0` to disable basic block filtering and rely only on `MINHASH_FN_MIN_INS`.
</ParamField>

## Signature Configuration

<ParamField path="MINHASH_SIGNATURE_LENGTH" type="int" default="64">
  Length of the MinHash signature in number of shingles.
  
  Larger signatures:
  - Provide more precise matching
  - Increase computational and storage costs
  - Should match banding configuration requirements
</ParamField>

<ParamField path="MINHASH_SIGNATURE_BITS" type="int" default="8">
  Number of bits per signature element (range: 1-32 bits).
  
  Common values:
  - `8` bits - Balanced precision and storage (default)
  - `16` bits - Higher precision
  - `32` bits - Maximum precision
</ParamField>

<ParamField path="MINHASH_SEED" type="int" default="0xDEADBEEF">
  Random seed used when initializing XOR values for MinHash seeds. Ensures reproducible hash generation.
</ParamField>

## Matching Configuration

<ParamField path="MINHASH_MATCHING_THRESHOLD" type="int" default="50">
  Lower bound percentage at which paired MinHashes are considered a match (range: 0-100).
  
  Higher values:
  - Require closer similarity for matches
  - Reduce false positives
  - May miss similar functions with variations
  
  Lower values:
  - Detect more similar functions
  - May increase false positives
</ParamField>

<ParamField path="BAND_MATCHES_REQUIRED" type="int" default="2">
  Minimum number of band matches a MinHash must have before being considered a candidate for detailed matching.
  
  This acts as a pre-filter to reduce the number of expensive similarity comparisons.
</ParamField>

## PicHash Configuration

PicHash provides exact matching for position-independent code hashes.

<ParamField path="PICHASH_SIZE" type="int" default="10">
  Minimum function size (in instructions) for considering PicHash matching.
</ParamField>

<ParamField path="PICHASH_IMPLIES_MINHASH_MATCH" type="bool" default="True">
  Do not perform MinHash matching for PicHash matches. Instead, assume PicHash matches automatically imply MinHash matches.
  
  <Note>
  This optimization skips redundant MinHash calculations when functions already match exactly via PicHash.
  </Note>
</ParamField>

## Multiprocessing

<ParamField path="MINHASH_POOL_INDEXING" type="bool" default="True">
  Enable multiprocessing during indexing operations.
  
  <Warning>
  When running as a server with Gunicorn/Falcon, multiprocessing may cause issues. Disable if you experience problems.
  </Warning>
</ParamField>

<ParamField path="MINHASH_POOL_MATCHING" type="bool" default="True">
  Enable multiprocessing during matching operations.
  
  <Warning>
  When running as a server with Gunicorn/Falcon, multiprocessing may cause issues. Disable if you experience problems.
  </Warning>
</ParamField>

## Performance Tuning

These parameters control batch sizes for processing operations to balance memory usage and performance.

<ParamField path="MINHASH_MATCHING_FUNCTION_BATCH_SIZE" type="int" default="10000">
  Number of functions to process per batch during candidate matching.
</ParamField>

<ParamField path="MINHASH_MATCHING_CANDIDATE_WORKPACK_SIZE" type="int" default="20000">
  Number of candidate pairs to process per work unit during matching.
</ParamField>

<ParamField path="MINHASH_GENERATION_WORKPACK_SIZE" type="int" default="10000">
  Number of functions to process into MinHashes per work iteration.
</ParamField>

<ParamField path="MINHASH_BAND_REBUILD_WORK_PACKAGE_SIZE" type="int" default="100000">
  Number of MinHash bands to process per package when rebuilding band indices.
</ParamField>

## Configuration Hash

The `MinHashConfig` class provides a `getConfigHash()` method that generates a SHA-256 hash of critical configuration parameters:

- `MINHASH_STRATEGY`
- `MINHASH_FN_MIN_INS`
- `MINHASH_FN_MIN_BLOCKS`
- `MINHASH_SIGNATURE_LENGTH`
- `MINHASH_SIGNATURE_BITS`
- `MINHASH_SEED`
- `PICHASH_SIZE`

This hash can be used to verify configuration consistency across different MCRIT instances.

## Example Configurations

### Default Configuration (Balanced)
```python
from mcrit.config.MinHashConfig import MinHashConfig

minhash_config = MinHashConfig()
# Uses defaults: 64-length signatures, 8-bit elements, 50% threshold
```

### High Precision Configuration
```python
minhash_config = MinHashConfig(
    MINHASH_SIGNATURE_LENGTH=128,
    MINHASH_SIGNATURE_BITS=16,
    MINHASH_MATCHING_THRESHOLD=70,
    BAND_MATCHES_REQUIRED=3,
    MINHASH_FN_MIN_INS=20
)
```

### Fast Performance Configuration
```python
minhash_config = MinHashConfig(
    MINHASH_SIGNATURE_LENGTH=32,
    MINHASH_MATCHING_THRESHOLD=40,
    BAND_MATCHES_REQUIRED=1,
    MINHASH_MATCHING_FUNCTION_BATCH_SIZE=50000,
    MINHASH_MATCHING_CANDIDATE_WORKPACK_SIZE=100000
)
```

### Server-Safe Configuration (Disable Multiprocessing)
```python
minhash_config = MinHashConfig(
    MINHASH_POOL_INDEXING=False,
    MINHASH_POOL_MATCHING=False
)
```

## Performance Considerations

1. **Signature Length**: Longer signatures improve accuracy but increase:
   - Storage requirements
   - Comparison time
   - Memory usage

2. **Matching Threshold**: Balance between:
   - False positives (threshold too low)
   - False negatives (threshold too high)

3. **Band Matching**: More required band matches:
   - Reduce candidate set size
   - Speed up overall matching
   - May miss similar functions

4. **Batch Sizes**: Larger batches:
   - Improve throughput
   - Increase memory usage
   - May cause issues on memory-constrained systems
