## Docker Quickstart (Recommended)

<Note>
We highly recommend using the fully packaged [docker-mcrit](https://github.com/danielplohmann/docker-mcrit) for trivial deployment and usage. This ensures fully compatible versions across all components, including a database for persistence and a web frontend for convenient interaction.
</Note>

The Docker deployment provides:
- Pre-configured MongoDB database
- MCRIT server and worker
- Web frontend for visualization
- All dependencies properly configured

Visit the [docker-mcrit repository](https://github.com/danielplohmann/docker-mcrit) for complete Docker setup instructions.

## Standalone Quickstart

For standalone installations, follow these steps to get MCRIT running quickly:

<Steps>
  <Step title="Start MongoDB">
    Ensure MongoDB is running on your system:
    
    ```bash
    sudo systemctl start mongod
    ```
    
    You can verify MongoDB is running:
    
    ```bash
    sudo systemctl status mongod
    ```
  </Step>

  <Step title="Start MCRIT Server">
    Launch the MCRIT server in a terminal:
    
    ```bash
    mcrit server
    ```
    
    By default, the REST API server will be listening on [http://127.0.0.1:8000/](http://127.0.0.1:8000/).
    
    <Note>
    Keep this terminal open. The server must be running to accept requests.
    </Note>
  </Step>

  <Step title="Start MCRIT Worker">
    In a new terminal, start a worker to process jobs:
    
    ```bash
    mcrit worker
    ```
    
    The worker will connect to the server and begin processing queued jobs for matching and analysis.
    
    <Note>
    You can run multiple workers in parallel for better performance on large datasets.
    </Note>
  </Step>

  <Step title="Verify Installation">
    Check the status of your MCRIT instance:
    
    ```bash
    mcrit client status
    ```
    
    You should see output similar to:
    
    ```json
    {
      'status': {
        'db_state': 187,
        'storage_type': 'mongodb',
        'num_bands': 20,
        'num_samples': 137,
        'num_families': 14,
        'num_functions': 129110,
        'num_pichashes': 25385
      }
    }
    ```
  </Step>
</Steps>

## Using the Python Client

The Python client provides programmatic access to all MCRIT functionality:

<CodeGroup>
```python Basic Connection
from mcrit.client.McritClient import McritClient

# Connect to MCRIT server
client = McritClient(mcrit_server="http://localhost:8000")

# Check server status
status = client.getStatus()
print(f"Connected to MCRIT with {status['num_samples']} samples")
```

```python With Authentication
from mcrit.client.McritClient import McritClient

# Connect with API token
client = McritClient(
    mcrit_server="http://localhost:8000",
    apitoken="your-api-token",
    username="analyst"
)

# Or set credentials after initialization
client.setApitoken("your-api-token")
client.setUsername("analyst")
```

```python Submit Sample
from mcrit.client.McritClient import McritClient

# Initialize client
client = McritClient()

# Submit a binary for analysis
with open("sample_binary", "rb") as f:
    binary_data = f.read()

# Submit with family name
result = client.addBinarySample(
    binary_data,
    filename="sample_binary",
    family="malware_family"
)

print(f"Sample ID: {result['sample_id']}")
print(f"Functions found: {result['num_functions']}")
```
</CodeGroup>

## Using the CLI

MCRIT provides a convenient command-line interface for quick operations:

<Steps>
  <Step title="Check Status">
    Query database statistics:
    
    ```bash
    mcrit client status
    ```
  </Step>

  <Step title="Submit a Sample">
    Submit a malware sample for analysis:
    
    ```bash
    mcrit client submit sample_unpacked -f some_family
    ```
    
    Output:
    ```
    1.039s -> (architecture: intel.32bit, base_addr: 0x10000000): 634 functions
    ```
  </Step>

  <Step title="Environment Variables">
    Configure server and authentication using environment variables:
    
    ```bash
    export MCRIT_CLI_SERVER="http://your-server:8000"
    export MCRIT_CLI_APITOKEN="your-api-token"
    
    mcrit client status
    ```
    
    Or create a `.env` file:
    ```
    MCRIT_CLI_SERVER=http://your-server:8000
    MCRIT_CLI_APITOKEN=your-api-token
    ```
  </Step>
</Steps>
